# NSCLC Multi-Cohort Clinical Database 

## Abstract
Данный репозиторий содержит программный комплекс для структурированного хранения, обработки и визуального анализа клинико-молекулярных данных пациентов с немелкоклеточным раком легкого (НМРЛ). Система реализует гибридную архитектуру базы данных, разделяющую общие клинические параметры и специфичные молекулярные профили (ROS1, ALK, EGFR, WT). Комплекс включает ETL-конвейер для автоматической нормализации сырых табличных данных и веб-интерфейс для интерактивного управления когортой и проведения аналитических запросов.

## Prerequisites
Для корректной работы проекта требуется:
- Python 3.12 или выше
- Операционная система: Linux (Ubuntu/WSL), macOS или Windows
- Менеджер пакетов `pip`

## Installation & Initialization

### 1. Клонирование репозитория
```bash
git clone https://github.com/Kvate/Database.git
cd Database
```
### 2. Установка зависимостей
Рекомендуется использовать виртуальное окружение:
```
python3 -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Обработка данных (ETL Pipeline)
Скрипт main.py выполняет автоматическое распознавание когорты по метаданным файла, анонимизацию персональных данных, нормализацию регистра и преобразование временных интервалов терапии в дни. Результатом работы является инициализированная база данных data/nsclc_cohorts.db.
Выполните запуск для каждого файла когорты:
```
python3 main.py data/ROS1.xlsx
python3 main.py data/ALK.xlsx
```
*При необходимости добавьте файлы EGFR и WT

**Примечание: Файл базы данных nsclc_cohorts.db генерируется автоматически после выполнения ETL-скрипта. 
### 4. Запуск веб-интерфейса
После успешной инициализации базы данных запустите аналитическую платформу:

```
streamlit run web_ver.py
```
Приложение откроется в браузере по адресу http://localhost:8501.

## Database Architecture
Проект использует реляционную СУБД SQLite с гибридной схемой, оптимизированной для онкологических регистров:

all_patients — централизованная таблица демографических и клинических параметров (возраст, пол, гистология, стадия, статус курения).

alk_fusion, ros1_fusion, egfr_mutation, wt — специализированные таблицы молекулярных профилей, связанные внешними ключами с all_patients.

treatments — история линий лекарственной терапии с указанием длительности в днях, препаратов и ответа по критериям RECIST.

outcomes — показатели выживаемости (PFS, OS) и статус цензурирования данных.

## Functional Modules
ETL Pipeline - Парсинг Excel, удаление PII, приведение регистра, расчет длительности терапии, маршрутизация по когортам

Dashboard - Агрегированная статистика, распределение по полу/возрасту/гистологии, визуализация длительности терапии

Data Management - Интерактивное редактирование таблиц, ручное добавление пациентов, экспорт в CSV

SQL Runner - Выполнение пользовательских SELECT-запросов с автоматической визуализацией результатов
